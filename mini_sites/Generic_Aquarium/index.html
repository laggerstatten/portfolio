Generic Aquarium

Inspiration

I'm working out of northern Alabama this week, and visited the Tennessee Aquarium in Chattanooga . I tend to visit a lot of zoos and aquaria, so naturally I make note of ones with good websites as well. The Tennessee Aquarium website is one of these.

I'm happy to note that the Tennessee Aquarium does have some animation effects on their horizontal nav bar (when viewed on a desktop). When the user hovers over a nav item (such as "plan a visit"), the nav item text changes color, slowly bounces a bit, and an effect occurs emanating from the cursor. I think this is a nice subtle but noticeable effect.

I spent way too long in Chrome trying to figure out how they accomplished this. I appreciate the work we're doing in this course, but the department really needs a "how to use Chrome developer tools to reverse-engineer a website" course. I think we've both been in the industry long enough to know that often our guidance is "the program manager for XYZ saw this on somebody else's website -- we need to do something similar but better on ours".

I don't think many of us are building websites from scratch. In my current role, I'm building "HubSites" within ESRI's ArcGIS Online environment. This is a low-code platform based on Bootstrap where I can choose to expose and edit the HTML and CSS (but not JS).

As I'm writing this, I decided to do a deep dive[1] and see who built the Tennessee Aquarium's website. It's Whiteboard out of Chattanooga, and there is even an interview on their design process.

I'll quote in full the last paragraph:

"'The website was built in WordPress while harnessing powerful tools, such as Advanced Custom Fields Pro, Timber (for Twig integration), Bedrock and Lumberjack,' says Brett Bash. 'This enables the WordPress CMS to have a modular, modern MVC framework structure similar to Laravel. The front end uses Vue.js for the JavaScript framework, GreenSock for animations and Tailwind CSS for all of the styling.'

My Design Process

For my design process, I started with the transitions and transformations, since they are the core of this week's assignment. I wanted an effect that would be very noticable on mouse-enter, but persist while the cursor was over the link. There are ways to do this with JS, but in CSS I had to do this by adding a second animation with a delay as long as the first animation. I made the link text bounce a bit and change color, and then continue subtly bouncing.

I went to Google Fonts and found 3 fonts recommended for use together. I used CSS to apply these to the different elements on the page. I used Envato Elements to find a few photos available for my use, and chose text colors that looked good over them.

On many of the websites I visit, the nav bar is placed over the hero photo. I tried to accomplish this with HTML and CSS, and spent way too long messing with position: absolute and z-index and a whole lot of other CSS properties. In reality, nobody is composing websites from scratch like this. In a real production environment, a new employee fresh out of school who is tasked with building a website is not going to succeed if they try building a website from scratch. They need to understand how to use the industry-standard / market-leader platforms that are in use. I appreciate that an understanding of web language fundamentals is beneficial, but I wonder if the primary focus should be on WordPress, knowing that web designers/developers will pick up HTML and CSS on the job.

For my navbar, I wanted it to start from the upper right corner. I applied padding-right to ensure the animation would not go off screen. I added a transparent background so that it would overlay the hero photo. I made the default color for the link text white, set the font size, and added padding above and below. I removed the underline. On hover, the link text turns red, and the transformation makes this a gradual change.

I originally made the navbar with just a div and a elements, so I had to go back and change it to use ul and li. This was likely an unneccesary requirement. I know earlier in the semester we discussed using semantic elements versus using divs (their use was derided as "divitis"). Go do an inspect on any professional website and see how many are using divs versus "section" and "article". Spoiler alert: it's "divs all the way down".

Neither technique from WCAG which addresses semantic elements mentions sections or articles: G115, G140. None of the three resources linked in Week 1 do a very good job explaining the "why" behind semantic elements, and in fact I think they are counterproductive. They seem to suggest that one could build a real modern public-facing website with all divs replaced by semantic elements. This would likely be even worse for accessibility, as now a screenreader is presented with dozens of unneccesary sections and articles coded by a well-meaning developer trying to avoid "divitis".

I did set links to open in new tabs. We just had this discussion at work, and the decision was against this, citing WCAG.

[1] The aquarium building itself is a very clever design as well -- it's from Cambridge Seven Associates.
